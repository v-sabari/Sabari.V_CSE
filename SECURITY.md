# Security Policy

This is a static personal portfolio site with no backend or user data
collection in the current build, so the attack surface is minimal. That said,
if you find a security issue (e.g. a dependency vulnerability or an XSS
vector), please report it responsibly.

## Reporting a vulnerability

Please **do not** open a public issue for security reports. Instead, email
sabarivenkatesan2962006@gmail.com with:

- A description of the issue and its potential impact
- Steps to reproduce
- Any suggested fix, if you have one

You should receive a response within a few days. Once the issue is confirmed
and fixed, a note will be added to the changelog.

## Supported versions

Only the latest version on the `main` branch is supported.
